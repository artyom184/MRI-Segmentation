## 3.1. Архитектура программного решения

### Технологии
| Компонент | Значение |
|-----------|----------|
| Язык программирования | Python 3.13 |
| Фреймворк глубокого обучения | PyTorch 2.11 + torchvision 0.26 |
| Постобработка | OpenCV 4.x, NumPy, SciPy (gaussian_filter1d) |
| Окружение | Терминальные скрипты `.py` |
| GUI | Нет (CLI-приложение) |

### Структура кода — 4 модуля
| Файл | Назначение |
|------|-----------|
| `pipeline_seg.py` | Конвертация DICOM + .seg.nrrd → обучающий датасет |
| `train_maskrcnn.py` | Обучение Mask R-CNN (ResNet-50 + FPN) |
| `deformity_analysis.py` | Алгоритм морфометрии: 6 точек → 3 высоты → Genant |
| `main.py` | Инференс + визуализация + текстовый отчёт |

### Конвейер обработки одного снимка (7 этапов)
1. **Загрузка** — чтение JPG 512×512
2. **Нормализация** — приведение к [0, 1]
3. **Инференс** — Mask R-CNN генерирует бинарные маски каждого позвонка
4. **Постобработка** — морфология OPEN + CLOSE + выделение наибольшего связного компонента
5. **Поворот** — компенсация лордоза (cv2.minAreaRect)
6. **Морфометрия** — нахождение 6 ключевых точек, расчёт 3 высот
7. **Классификация** — определение степени деформации по Genant → отчёт (JPG + текст)

### Форматы данных
- **Вход**: DICOM-серии (при подготовке датасета), JPG 512×512 (при инференсе)
- **Разметка**: .seg.nrrd (3D Slicer) → бинарные маски
- **Выход**: аннотированное изображение JPG (6 точек + 3 линии высот) + текстовый отчёт

## 3.3. Обучение модели

### Архитектура Mask R-CNN
| Компонент | Значение |
|-----------|----------|
| Архитектура | **Mask R-CNN** |
| Backbone | **ResNet-50** |
| Neck | **Feature Pyramid Network (FPN)** |
| Предобучение backbone | ImageNet |
| Предобучение всей модели | **COCO** (transfer learning) |
| Общее число параметров | **43 975 515** (~44 млн) |
| Размер весов | **170 MB** |
| Фреймворк | PyTorch 2.11 / torchvision 0.26 |

### Функция потерь
L = L_cls + L_box + L_mask (формула 1 из статьи [10])
- L_cls — classification loss (кросс-энтропия)
- L_box — bounding-box regression loss (Smooth L1)
- L_mask — binary cross-entropy loss для масок

### Гиперпараметры обучения
| Параметр | Значение |
|----------|----------|
| Input size | **512×512** |
| Batch size | 2 |
| Оптимизатор | **SGD** |
| Learning rate | **0.005** |
| Momentum | 0.9 |
| Weight decay | 0.0005 |
| LR scheduler | StepLR (step=20, gamma=0.5) |
| Max detections per image | 8 |
| Число классов | 2 (фон + позвонок) |
| Early stopping patience | 10 |
| Всего эпох | **30** |
| Лучшая эпоха | **28** |

### Кривая обучения
Модель успешно сошлась за 30 эпох.
- **Начальные значения (Эпоха 1):** Train loss ≈ 1.32, Val loss ≈ 0.65
- **Лучшие значения (Эпоха 28):** Train loss = **0.1512**, Val loss = **0.1984**
- **Итоговые значения (Эпоха 30):** Train loss = 0.1490, Val loss = 0.2011

> Train loss плавно убывает на протяжении всех 30 эпох. Val loss достигает минимума в районе 28 эпохи и далее стабилизируется. Разница между Train и Val loss остается минимальной, что свидетельствует об отсутствии сильного переобучения и хорошей обобщающей способности модели. Графики обучения сохранены в `results_maskrcnn/loss_curve_full.png` и `results_maskrcnn/pr_curve.png`.

## 3.4. Алгоритм морфометрического анализа

### Соответствие статье
Воспроизводит метод из **Section 2.5** статьи [10].
Реализован в `deformity_analysis.py` (v2).

### Библиотеки
NumPy + SciPy (gaussian_filter1d) + OpenCV

### Шаги алгоритма
1. **Постобработка маски** (Section 2.5.1): OPEN (5×5) → CLOSE (5×5, 2 итерации) → largest connected component
2. **Поворот** (Section 2.5.1): cv2.minAreaRect → поворот вокруг центра масс
3. **Разделение** (Section 2.5.2): верх/низ по Y центра масс
4. **Гистограмма** (Section 2.5.2): подсчёт ненулевых пикселей по X → гауссово сглаживание (σ = 3% ширины)
5. **6 точек** (Section 2.5.2, Fig. 6):
   - Au, Al — максимум в первых 10%
   - Pu, Pl — максимум в последних 10%
   - Mu, Ml — минимум в ±10% от центра масс
6. **3 высоты** (Section 2.5.3, формулы 4–6): Ah, Ph, Mh

### 3 авторские модификации
| # | Статья [10] | Наша реализация | Причина |
|---|---|---|---|
| 1 | PCA для угла | **cv2.minAreaRect** | Стабильнее на краевых масках |
| 2 | Без сглаживания | **Гауссово сглаживание** (σ = 3%) | Устраняет пиксельный шум |
| 3 | OPEN + CLOSE | + **largest connected component** | Убирает мусор от соседних масок |

## 3.5. Классификация деформаций по Genant

### Формулы (Section 2.5.4)
- Если Ah < Ph: APR = 100−(Ah/Ph)×100; MPR = 100−(Mh/Ph)×100
- Если Ph ≤ Ah: PAR = 100−(Ph/Ah)×100; MAR = 100−(Mh/Ah)×100

### Пороги
| Grade | Снижение | Описание |
|-------|----------|----------|
| 0 | < 20% | Normal |
| 1 | 20–25% | Mild |
| 2 | 25–40% | Moderate |
| 3 | > 40% | Severe |

### Типы деформации
- Wedge: APR или PAR ≥ 20%
- Biconcave: MPR или MAR ≥ 20%
- Итоговый Grade = max(wedge_grade, biconcave_grade)
